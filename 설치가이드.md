# 🌸 민성이의 하루 - 설치 가이드

민성님, 나비입니다! 아래 순서대로 천천히 따라오시면 됩니다.
전체 소요시간은 약 **20~25분**이고, 크게 4단계입니다.

> 1단계 Firebase 만들기 → 2단계 설정값 붙여넣기 → 3단계 GitHub 올리기 → 4단계 텔레그램 알림 연결

---

## 1단계. Firebase 프로젝트 만들기 (약 10분)

기존에 운동앱·NMS 만들 때 하셨던 것과 같은 과정입니다.

### 1-1. 프로젝트 생성
1. 크롬에서 https://console.firebase.google.com 접속 → 구글 계정 로그인
2. **"프로젝트 추가"** 클릭
3. 프로젝트 이름: `minsung-day` 입력 → 계속
4. Google 애널리틱스: **"사용 안함"** 으로 끄고 → **프로젝트 만들기** → 완료까지 대기

### 1-2. 로그인 기능 켜기 (Authentication)
1. 왼쪽 메뉴 **빌드 → Authentication** 클릭 → **시작하기**
2. **"이메일/비밀번호"** 클릭 → 첫 번째 스위치 **사용 설정** 켜기 → 저장

### 1-3. 데이터베이스 만들기 (Firestore)
1. 왼쪽 메뉴 **빌드 → Firestore Database** 클릭 → **데이터베이스 만들기**
2. 위치: **asia-southeast1 (싱가포르)** 선택 → 다음
3. **"프로덕션 모드"** 선택 → 만들기
4. 만들어지면 위쪽 **"규칙(Rules)"** 탭 클릭 → 내용을 전부 지우고 아래를 붙여넣기 → **게시** 클릭

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
    }
  }
}
```

> 💡 이 규칙은 "본인 계정으로 로그인한 사람만 본인 데이터를 볼 수 있다"는 뜻입니다. 보안상 꼭 필요합니다.

### 1-4. 설정값 복사하기
1. 왼쪽 위 **⚙️(톱니바퀴) → 프로젝트 설정** 클릭
2. 아래로 내리면 **"내 앱"** → **`</>` (웹)** 아이콘 클릭
3. 앱 닉네임: `minsung-day` 입력 → **앱 등록** (호스팅 체크는 안 하셔도 됩니다)
4. 화면에 나오는 `const firebaseConfig = { ... }` 부분을 **복사**해 두세요 (메모장에 임시 저장)

## 2단계. index.html에 설정값 붙여넣기 (약 2분)

1. 제가 드린 `index.html` 파일을 메모장(또는 VS Code)으로 열기
2. 파일 위쪽에서 이 부분을 찾으세요:

```
const firebaseConfig = {
  apiKey: "여기에_붙여넣기",
  ...
};
```

3. 1-4에서 복사한 값으로 **통째로 바꿔치기** → 저장

---

## 3단계. GitHub에 올리고 주소 받기 (약 5분)

1. https://github.com 로그인 → 오른쪽 위 **+ → New repository**
2. 이름: `minsung-day` → **Public** 선택 → **Create repository**
3. **"uploading an existing file"** 링크 클릭
4. 제가 드린 파일들을 **폴더 구조 그대로** 끌어다 놓기:
   - `index.html`
   - `reminder.js`
   - `.github/workflows/reminder.yml` ← ⚠️ 폴더째로 올려야 합니다!
   > 💡 폴더째 업로드가 안 되면: ZIP을 컴퓨터에서 풀고, 풀린 폴더 안의 내용물 전체를 선택해서 GitHub 업로드 창에 드래그하세요.
5. 아래 **Commit changes** 클릭
6. 저장소에서 **Settings → Pages** (왼쪽 메뉴) 클릭
7. Branch: **main** 선택, 폴더: **/(root)** → **Save**
8. 1~2분 뒤 위쪽에 주소가 나옵니다:
   `https://아이디.github.io/minsung-day/`
   → 이게 앱 주소입니다! 노트북·휴대폰 즐겨찾기에 등록하세요 🌸

### 첫 사용
- 주소 접속 → **"처음이에요 (가입)"** 버튼으로 이메일+비밀번호(6자 이상) 가입
- 노트북과 휴대폰에서 **같은 이메일로 로그인**하면 데이터가 자동 동기화됩니다
- 📱 휴대폰 팁: 크롬에서 열고 메뉴 → **"홈 화면에 추가"** 하면 앱처럼 아이콘이 생깁니다

## 4단계. 텔레그램 저녁 8시 알림 연결 (약 8분)

브라우저를 꺼놔도 매일 **베트남시간 저녁 8시**에 미완료 할일을 텔레그램으로 보내줍니다.

### 4-1. 텔레그램 봇 만들기
1. 텔레그램에서 **@BotFather** 검색 → 대화 시작
2. `/newbot` 입력
3. 봇 이름 입력: `민성이의하루` (아무거나 OK)
4. 봇 아이디 입력: `minsungday_bot` 처럼 **bot으로 끝나는** 영문 아이디 (이미 있으면 다른 이름으로)
5. 성공하면 **토큰**이 나옵니다. 예: `7123456789:AAHxxx...`
   → 복사해서 메모장에 저장 (= `TG_TOKEN`)

### 4-2. 내 채팅 ID 알아내기
1. 방금 만든 봇을 검색해서 대화창 열고 **/start** 를 꼭 한 번 보내기
2. 크롬 주소창에 아래를 입력 (토큰 부분만 바꿔서):
   `https://api.telegram.org/bot토큰붙여넣기/getUpdates`
3. 화면에 나오는 글자 중 `"chat":{"id":123456789` 에서 **숫자**가 채팅 ID입니다
   → 메모장에 저장 (= `TG_CHAT_ID`)

### 4-3. GitHub에 비밀값 등록 (Secrets)
1. GitHub 저장소 → **Settings → Secrets and variables → Actions** 클릭
2. **New repository secret** 버튼으로 아래 **6개**를 하나씩 등록:

| 이름 (Name) | 값 (Secret) |
|---|---|
| `FIREBASE_API_KEY` | firebaseConfig의 apiKey 값 |
| `FIREBASE_PROJECT_ID` | firebaseConfig의 projectId 값 |
| `FB_EMAIL` | 앱 가입할 때 쓴 이메일 |
| `FB_PASSWORD` | 앱 가입할 때 쓴 비밀번호 |
| `TG_TOKEN` | 4-1의 봇 토큰 |
| `TG_CHAT_ID` | 4-2의 숫자 |

> 🔒 Secrets는 GitHub이 암호화해서 보관하므로 남에게 안 보입니다. **절대 index.html 안에는 토큰을 넣지 마세요** (공개 저장소라 노출됩니다).

### 4-4. 알림 테스트
1. 저장소 위쪽 **Actions** 탭 클릭
2. 왼쪽 **"🔔 저녁 8시 텔레그램 알림"** 클릭 → 오른쪽 **Run workflow → Run workflow** 버튼
3. 30초쯤 뒤 초록 체크✅가 뜨고, 미완료 할일이 있으면 텔레그램 메시지가 옵니다!
   (미완료 할일이 없으면 "알림 생략"으로 정상 종료됩니다 → 할일 하나 등록하고 다시 테스트해 보세요)

## ⚠️ 알아두실 점 (정직하게 말씀드립니다)

1. **웹 화면 알람**: 브라우저에 앱이 열려 있을 때만 저녁 8시에 팝업이 뜹니다. 닫혀 있으면 → 텔레그램 알림이 대신 갑니다.
2. **GitHub 알림 시간**: GitHub 예약 실행은 몇 분(길면 10~20분) 늦어질 수 있습니다. 무료 서비스의 특성입니다.
3. **비용**: Firebase 무료 등급 + GitHub 무료로 충분합니다. 개인 사용량으로는 돈이 들지 않습니다.

## 🆘 문제가 생기면

- 로그인이 안 돼요 → 1-2단계(이메일/비밀번호 사용 설정) 확인
- 데이터 저장이 안 돼요 → 1-3단계 규칙(Rules) 붙여넣기 확인
- 텔레그램이 안 와요 → Actions 탭에서 빨간 ❌ 클릭하면 오류 내용이 보입니다. 캡처해서 나비에게 보여주세요!
